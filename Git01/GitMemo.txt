깃 - 버전 관리를 위한 CUI 프로그램
소스트리 - 깃을 더욱 쉽게 조작할 수있도록 GUI로 설계한 프로그램
깃허브 - 깃으로 버젼 관리한 이전 버젼들을 인터넷에 저장할수있도록 만든 클라우드 서비스

작업 디렉터리	:	제작/수정/삭제한 소스코드가 저장된 장소
작업 디렉터리의 작업물을 스테이지로 올리는 작업을 add라고 합니다.

스테이지		:	작업 디렉터리에서 만진 코드들 중 어느것을 저장소에 올릴지 임시적으로 거쳐가는 공간
commit  : 스테이지의 내용을 저장소에 올리는 것(확정)을 commit이라고 합니다.

저장소		:	새로운 버젼 저장하는 공간을 저장소	
add : 스테이지에 올린 작업물을 저장소에 올리는 작업


태그는 프로젝트의 업데이트 버젼을 붙이는것을 권장합니다
(ex. Ver1.0.2 >> 태그 Ver0.0.2)

Python 문법

1. 기본적으로 타입정의가 필요가 없습니다

2. 함수를 정의하기 위해서는 아래의 형식을 따릅니다.

	def 함수이름(참조 변수) : 함수 내용 return 결과값


3. 세미콜론은 필수적으로 사용하지 않습니다.
